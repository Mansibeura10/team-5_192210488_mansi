STEP-BY-STEP SETUP IN IICS
1
WRAP THE MAPPING IN A MAPPING TASK (MCT)

In IICS, a Taskflow cannot run a raw Mapping directly — it must be wrapped first.

  • Go to New > Tasks > Mapping Task
  • Name it: mct_CallType_Performance
  • Select Mapping: Choose your Source → Aggregator → Expression → Target flow
  • Runtime Environment: Select your Secure Agent
  • Click Finish
2
BUILD THE TASKFLOW CANVAS

Create the Taskflow that orchestrates the mapping execution with decision logic.

  • Go to New > Taskflows > Taskflow > Create
  • Name it: tf_CallType_Performance_Job
  • Canvas opens with a Start node and End node
  • Drag a Data Task step between Start and End — name it Run_Performance_Mapping
  • In the Data Task tab, click Select → choose mct_CallType_Performance
3
ADD DECISION STEP — ERROR HANDLING

After the Data Task, add a Decision step to evaluate success or failure.

  • Drag a Decision step after the Data Task — name it Check_Success
  • Create condition: Run_Performance_Mapping → Task Status = 1 (Success)
  • Path 1 (Success): Connect directly to the End node
  • Otherwise (Failure): Drag a Notification Task onto this path to send an email alert
  • Connect the bottom of the Notification Task → End node
4
SAVE AND PUBLISH

Finalize and publish the Taskflow to make it available for scheduling and execution.

  • Click Save in the top right corner
  • Click Publish to deploy the Taskflow to the IICS environment
